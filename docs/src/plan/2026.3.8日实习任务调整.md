### 为文件系统增加通用日志模块

**目标**：
1. 设计并实现一套通用的文件系统日志框架
3. 记录关键操作（读写）和性能指标例如延迟、吞吐量（额外feature）
4. 可以在使用任何除了ramfs外的实际操作系统上保证数据不会丢失


# 初步设计
- 设计一个独立的文件系统日志包，在虚拟文件系统将实际读写操作分发具体文件系统前将具体的分发代码改为文件系统日志包。 文件系统日志包内部再进行具体的文件系统分发，读写功能与原来无异，接口也不会变得复杂。
- 文件系统日志包内部在写入时进行crc32c校验和日志记录，然后写入实际磁盘。 读取时进行crc32c校验，然后返回给上层调用者。

- 文件系统日志包内部具体接口有如下设置:
- 宏设计： 外部代码只需要套上日志宏，宏内部会生成日志代码
## 例如原来的代码
``` rust
pub fn write_at(&self, buf: impl Read + IoBuf, offset: u64) -> VfsResult<usize> {
        let _guard = self.append_lock.read();
        self.write_at_locked(buf, offset)
}
```
## 变为
``` rust
pub fn write_at(&self, buf: impl Read + IoBuf, offset: u64) -> VfsResult<usize> {
        use axfs_log::journal_write;
        journal_write!{
          let _guard = self.append_lock.read();
          self.write_at_locked(buf, offset);
        }   
}
```
- 然后后续宏展开生成具体日志代码,并且可以省去一层函数调用的性能损耗
